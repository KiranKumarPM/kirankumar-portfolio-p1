### 1. Project Structure

Your project is a standard React application built with Vite. Here’s a breakdown of the key files and folders:

*   **`index.html`**: The main HTML file. Your entire React application is injected into the `<div id="root"></div>` element inside this file.
*   **`package.json`**: This file lists all your project's dependencies (like React, Framer Motion, and Tailwind CSS) and defines scripts to run, build, and test your application.
*   **`vite.config.js`**: The configuration file for Vite, your project's build tool. It tells Vite how to compile and serve your application.
*   **`tailwind.config.cjs` & `postcss.config.cjs`**: These files configure Tailwind CSS, your styling framework.
*   **`public/`**: A folder for static assets like images, videos, and certificates that are directly accessible in the browser.
*   **`src/`**: This is where all your source code lives.
    *   **`main.jsx`**: This is the **entry point** of your application. It finds the `root` div in `index.html` and tells React to render your main `App` component inside it.
    *   **`index.css`**: A global stylesheet where you've defined custom styles, including the animated cursor, background video styling, and link hover effects.
    *   **`App.jsx`**: The main component that acts as a container for your entire portfolio. It assembles all the different sections (Hero, Projects, etc.).
    *   **`components/`**: This folder contains reusable UI pieces.
        *   `Navbar.jsx`: The navigation bar at the top of the page.
        *   `ProfileDropdown.jsx`: The dropdown menu in the navbar.
    *   **`sections/`**: This folder contains the major content blocks of your portfolio. Each file represents a different section of your single-page application (e.g., `Hero.jsx`, `Projects.jsx`, `Contact.jsx`).

### 2. Component Flow

Your application is built from nested components. The parent-child relationships are straightforward:

*   **`App.jsx`** is the top-level parent component.
*   Inside `App.jsx`, you render several child components in a specific order:
    1.  `Navbar`
    2.  `Hero`
    3.  `Education`
    4.  `Projects`
    5.  `TechnicalSkills`
    6.  `Certificates`
    7.  `GitHub`
    8.  `LinkedIn`
    9.  `LeetCode`
    10. `Contact`
*   The `Navbar` component itself has a child: `ProfileDropdown`.

Think of it like building blocks: `App.jsx` is the main board, and you place each section component onto it one by one.

### 3. Rendering Flow

Here is how your portfolio appears on the screen:

1.  The browser loads `index.html`.
2.  The script tag in `index.html` executes `src/main.jsx`.
3.  `main.jsx` renders the `App` component.
4.  The `App` component renders its children in sequence, from top to bottom. This is why the "Hero" section appears first, followed by "Education," "Projects," and so on, creating the vertical layout of your portfolio.

### 4. Data Flow

Your portfolio's data is handled in a simple way:

*   **Hardcoded Data**: Almost all the data (your education history, project details, skills list) is "hardcoded" directly inside the component that displays it. For example, the project information is written directly into the `Projects.jsx` file.
*   **Component State**: Some components use React's `useState` hook to manage internal data, like whether a dropdown menu is open or closed. This data is not shared with other components.
*   **Props**: You are not passing data down from parent to child components (using props) in most cases. The structure is simple, with each section managing its own content.

### 5. Styling System

You use a combination of three methods for styling:

1.  **Tailwind CSS**: This is your primary styling method. You use utility classes (like `bg-gray-900`, `text-white`, `p-4`) directly in your JSX to style elements. These are configured in `tailwind.config.cjs`.
2.  **Custom CSS**: For more complex styles that are hard to achieve with Tailwind alone (like the custom cursor and specific animations), you have written custom CSS rules in `src/index.css`.
3.  **Inline Styles**: Some dynamic styles, particularly for animations managed by Framer Motion, are applied as inline styles directly on the HTML elements.

### 6. Routing

Your portfolio is a **Single-Page Application (SPA)**.

*   You are **not using a routing library** like React Router.
*   Navigation is handled by standard HTML anchor links (e.g., `<a href="#projects">`). When a user clicks a link in the navbar, the browser smoothly scrolls to the corresponding section on the same page.

### 7. Projects Section Deep Dive

The "Projects" section works as follows:

1.  **Static Data**: Inside `Projects.jsx`, you have an array of JavaScript objects. Each object represents one project and contains its title, description, image, and links.
2.  **Mapping and Rendering**: The component uses the `.map()` function to loop through this array. For each project object in the array, it renders a "project card" component with that project's data.
3.  **Static Rendering**: Because the data is hardcoded, the project cards are static. To add, remove, or change a project, you would need to edit the array inside the `Projects.jsx` file directly.

### 8. Interactivity

Your portfolio has several interactive elements, primarily handled by **Framer Motion** and CSS:

*   **Animations**: Sections and elements fade in and slide up as you scroll down the page. This is done with Framer Motion's `whileInView` prop.
*   **Hover Effects**: Many elements react when you hover over them (e.g., buttons scale up, links show an underline, project cards tilt). These are created with a mix of CSS `:hover` pseudo-classes and Framer Motion's `whileHover` prop.
*   **Buttons and Links**: Standard `<a>` and `<button>` tags are used for navigation and actions. Their functionality is based on default browser behavior (scrolling to an anchor, opening a link) or simple JavaScript click handlers.
*   **Custom Cursor**: The animated dot that follows your mouse is created with JavaScript that tracks the mouse's position and updates the `top` and `left` CSS properties of two `div` elements.
